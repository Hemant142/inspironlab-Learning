Encapsulation in Angular

This component demonstrates the use of encapsulation in Angular. Encapsulation is a way to control the visibility and accessibility of component styles and templates. In this example, we use the ViewEncapsulation enum to set the encapsulation mode for the component.

The encapsulation mode can be set to:

  • ViewEncapsulation.Emulated (default)
  • ViewEncapsulation.None
  • ViewEncapsulation.ShadowDom
  • ViewEncapsulation.Native

In this example, we use ViewEncapsulation.Emulated, which means that Angular will emulate the behavior of Shadow DOM by adding unique attributes to the component's styles. This allows styles to be scoped to the component without using actual Shadow DOM.

You can see the effect of encapsulation by inspecting the styles applied to this component in the browser's developer tools. The styles will be scoped to this component only, preventing them from affecting other components in the application.

Example :---Encapsulation:ViewEncapsulation.Emulated

  • In this inside the Child component we are doing Encapsulation:ViewEncapsulation.Emulated (encapsulation.component.ts) because of that the parent component (app-encapsulation-emulated) not getting red
  • In this example, we are using Emulated encapsulation, which means that the styles defined in the child component will not affect the parent component or other components.
